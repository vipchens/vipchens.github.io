Hexo3.2.2+Github我走的弯路

发表于 2016-08-16   |  

搭建这个Blog的时候，主要是参考这篇文章：HEXO搭建个人博客

参照上面这篇Blog大部分的命令都是可行的，但可能由于是版本变迁，有些命令现在的3.X版本不适用，我在这个过程中折腾了好几个晚上才弄明白！故把一些有变化的地方记录下来，一是自己备份，而是抱着能帮到别人的侥幸心理，废话不多说。

首先看下我的环境，Node.js版本为稳定版的4.4.7。

hexo: 3.2.2
hexo-cli: 1.0.2
os: Windows_NT 6.1.7601 win32 x64
http_parser: 2.5.2
node: 4.4.7
v8: 4.5.103.36
uv: 1.8.0
zlib: 1.2.8
ares: 1.10.1-DEV
icu: 56.1
modules: 46
openssl: 1.0.2h

配置SSH-Key

一开始我一直都是使用参考文章那样配置，但当我看到另一篇Blog的时候，才发现应该是这样的，正确性如何不知道，但我这么设置确实能Hexo d上。

1.生成指定名字的密钥(会生成github_jslite 和 github_jslite.pub 这两个文件在默认的位置：C:\Users\Administrator.ssh)

ssh-keygen -t rsa -C "邮箱地址"

2.打开公钥文件 id_rsa.pub ，并把内容复制至代码托管平台上

配置_config.yml（这是个折腾的我快放弃的地方）

1.打开文件找到最下面的这个部分：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type:

2.修改成这样的格式：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type: git
repository: git@github.com:vipchens/vipchens.github.io.git
branch: master

其中type: git 这个参数，在HEXO3.X后，已经由github变成了git,这个地方让我纠结了很久，直至快要放弃的时候才考虑到是不是版本的问题而去百度。
repository:git@github.com:vipchens/vipchens.github.io.git这个参数的方式如下图：

切记，修改HEXO所有参数的时候，参数与值之间一定要加上空格，一定要加上空格，一定要加上空格，重要的事情说三遍。（参数与值这个描述是我一个文科生对代码的理解，具体对不对，我觉得不重要，自己理解就行了）

hexo d 部署到Github

参考文章：Hexo 使用中遇到的问题总结
这个是在正常hexo d后，部署提示找不到git，是淫威在Hexo 3.0版本后deploy git 被分开的，所以需要安装，安装命令如下:

npm install hexo-deployer-git --save

安装好后再尝试hexo d一下就ok。

总结

1.安装环境的时候，需多尝试横向思考，别在一个问题或者一片文章中打转，正确的解决方法应该是把出错的结果找出来，多百度谷歌，有资源的可以多请教别人。像我这次就是，明明所有的步骤都是按照人家的来，为什么人家的可以，我的却是这里出错那里出错，教训之一。

2.作为一个只会使用图形界面的文科生第一次尝试在CMD下敲命令行产生静态Blog并部署到Github上成功的那一刻，内心有多兴奋，你们这些大神是无法理解的。

3.搭建Blog只是开始，坚持的学习、记录、分享才是最难的。

为何尝试写Blog，因为懒惰

发表于 2016-08-15   |  

因

2016年7月的某个夜晚，结束了一把lol之后，开始怀疑人生，虽然此前已经有无数次怀疑过，但都没有那次那么重过。

不知道从什么时候开始，有点过度的成迷了，也不知道从什么时候开始对胜败看的清淡了。也需是怕输，所以一开始就输了！每每想到这些都远厌倦自己，以前那个有一个兴趣就有无限动力的自己已经慢慢的消失了！

那晚躺在床上，翻来覆去都在想如何才能改变自己。思来复去总结得来，是因为自己懒、惰。

为何是懒、惰，而不是懒惰，以前的自己是懒，现在已经演变成惰,对任何事情提不起兴趣，有了兴趣遇到一点挫折就早早的放弃了。屋子懒得收拾，衣服堆在一起好几天洗一次，每天下班回家完全没有学习的欲望。但心里却实实在在的痛恨自己，每天都活在自责之中。

果

So，怀念起在前设计公司呆的日子，虽然项目不多，但每天都能逼着自己学更多的东西，每天都是如此的充实，也因为那时有一帮不服输的同事，谢谢你们！

我想找个东西每天警示着我要不停的进步，我想写Blog可能是一个不错的方法，可能从中能结识到志同道合的朋友，也能迫使我自己不断的学习，不断的进步！所以想到了，就着手开始准备去弄吧！

做

半年前，我在上任公司开始自学HTML、CSS，也做过一些小型的网站，虽然有些是套模板，有些是改样式、改布局什么的。但也接触了几个不错的建站程序，如：DEDE（织梦）、Wordpress、Duxcms。

dede是我觉得功能齐全，但使用特别复杂的程序，也可能是我对程序不大了解，很多细小的功能都不大完善。印象最深的是文章的排序和列表分页功能居然不能正常使用，当然我还是得承认也许是我调用模板的方式不对，或者是对程序不了解，百度了很久才解决这个问题。

Duxcms的故事比较纠结，以后再分享，总结是个不错的、简便的程序，现有1.1、2.0、3.0开发板，推荐1.1（因为稳定、有中英文站自由切换，企业站非常适合）

Wordpress是我用的比较有感情的程序。我用它弄过游戏攻略站、淘宝客站（这些两个当时都是找的免费模板，当时还没有能力自己写模板），近期也用它给公司搭建了一个电商Coupon Code分享博客，模板是自己写的，丑事丑了点，但总归还是学到不少东西，也第一次学会了使用插件给程序加新字段。

这次打算写Blog本打算也用Wordpress来搭建的，但无意中在喜马拉雅中听了一个节目内核恐慌，里面的静态网站生成器吸引了我。刚开始我以为只是伪静态的意思，但听了之后才发现原来有了静态博客的概念了。

百度了很久最终选择了Hexo+Github来搭建了这个Blog，对一个文科生而言，其中的艰辛和绕的圈子这里就不说了，后面我督促自己详细的写出来，不为别的，留个纪念也好。

至此，Blog算是搭建好了，慢慢的学习使用Markdown标记语言，这算是第一个尝试吧，希望开了Blog能不断的警醒自己不断的学习及进步！
